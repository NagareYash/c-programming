# Multi-dimensional Arrays and Memory Layout

## Pointers

All variables in a program are stored at a particular memory location. A pointer stores the memory address of another variable, giving you indirect access to data via its address.

**Example:**

```c
int n = 1;
int *pointer;

pointer = &n;       // associate pointer with n — ALWAYS do this before use!
n++;                // n is now 2
(*pointer)++;       // dereference pointer and increment — n is now 3!
```

### Key Operators

| Operator | Name        | Meaning                                      |
| -------- | ----------- | -------------------------------------------- |
| `&`      | Address-of  | Returns the **memory address** of a variable |
| `*`      | Dereference | Returns the **value at** a memory address    |

### Golden Rules

- **Always associate** a pointer with a variable before using it — an uninitialized pointer points to an unknown memory location, which is dangerous​
- Use `&var` to get the address, `*ptr` to get the value
- Pointers are especially useful for **sorting**, you only need to move addresses around, not the actual data​

## Arrays

Arrays group multiple values of the **same type** under one name, stored in **contiguous memory**. The brackets `[]` tell the compiler it's an array, and the constant inside specifies its size.

### Key Rules

```c
int a[100];     // allocates contiguous space for 100 ints
                // valid indices: a[0] to a[99]
```

- Elements are numbered from **0 to size-1**
- The compiler allocates **contiguous memory**, all elements sit side by side​
- `a` represents the **address (pointer) to the first element** of the array
- The computer knows where the **next element is** because every element has the same size, and the program knows that size from the **data type**

### How the Computer Knows Where Each Element Is located?

1. Element Size Is Known
   When you declare an array, the compiler knows the type and therefore the size:

```c
int A[5];   // Type = int, Size = 4 bytes   
```

1. Memory Layout
   If the base address is `1000`, memory looks like:

| Element | Address |
| :-----: | :-----: |
| `A[0]`  |  1000   |
| `A[1]`  |  1004   |
| `A[2]`  |  1008   |
| `A[3]`  |  1012   |
| `A[4]`  |  1016   |

1. General Formula
   The computer calculates each element's address using: `Address(A[i])=BaseAddress+(i×SizeOfType)`
   **Example:**
      - Base = `1000`, Size of `int` = `4`, `i = 3`
      - Address(`A[3]`) = `1000 + (3 × 4)` = `1012`

2. Pointer Arithmetic

```c
int *p = A;   // p points to first element — same as A
```

   `p + 1` does **not** mean `+1 byte` — it means `+sizeof(int)`:

```text
p + 0 → 1000
p + 1 → 1004
p + 2 → 1008
```

   The compiler **automatically multiplies** by the size of the type.

1. Element Size Varies by Type
   **`char` array — 1 byte each:**

```c
char A[5];
Address: 1000  1001  1002  1003  1004
```

   **`double` array — 8 bytes each:**

```c
double A[5];
Address: 1000  1008  1016  1024  1032
```

### Three Types of Initialization

```c
float f[3]  = {0.0, 1.0, 2.0};   // explicitly initialized
float f2[3] = {0};                // all elements set to 0
float f3[3];                      // ⚠️ uninitialized — garbage values!
```

## Multidimensional Arrays

C supports arrays of **arbitrary dimensions**. A two-dimensional array is essentially an **array of arrays**, each row is itself a 1D array.​

### Declaration

```c
int a[3][5];    // 2D array — 3 rows, 5 columns
```

- The compiler reserves `rows × columns` contiguous integer storage locations​
- `int a[3][5]` → `3 × 5 = 15` contiguous integers in memory

**Visualizing a 2D Array:**

```c
         Col0  Col1  Col2  Col3  Col4
        ┌─────┬─────┬─────┬─────┬─────┐
Row 0 → │     │     │     │     │     │
        ├─────┼─────┼─────┼─────┼─────┤
Row 1 → │     │     │     │     │     │
        ├─────┼─────┼─────┼─────┼─────┤
Row 2 → │     │     │     │     │     │
        └─────┴─────┴─────┴─────┴─────┘
```

### Initialization

**Initialize to zero:**

```c
int a[2][3] = {0};          // all elements set to 0
```

**Flat list: compiler fills row by row:**

```c
int a[2][3] = {1, 2, 3, 4, 5, 6};
// Row 0 → {1, 2, 3}
// Row 1 → {4, 5, 6}
```

**Clearer: using nested braces (preferred):**​

```c
int a[2][3] = {{1, 2, 3},
               {4, 5, 6}};
```

**Trailing zeros: compiler fills rest with 0:**

```c
// {2, 0, 0} can be written as {2}
// but always be explicit — easier to read and less error prone!
```

### 3D Arrays

Extends naturally to three dimensions:​

```c
int arr[3][3][3] = {
    { {11, 12, 13}, {14, 15, 16}, {17, 18, 19} },
    { {21, 22, 23}, {24, 25, 26}, {27, 28, 29} },
    { {31, 32, 33}, {34, 35, 36}, {37, 38, 39} }
};
```

- `int a[i][j][k]` → compiler sets aside `i × j × k` contiguous storage locations​
- Can be extended to **N dimensions** — same principles apply

### Memory Layout Row Major Order

Even though a 2D array looks like a grid, C stores it as a **single flat block** of memory in **row major order** — row 0 first, then row 1, then row 2:​

```shell
int a[2][3] = {{1,2,3},{4,5,6}};

Memory → [ 1 ][ 2 ][ 3 ][ 4 ][ 5 ][ 6 ]
           ←── Row 0 ───→←── Row 1 ───→
```

Address of `a[i][j]` is calculated as: `Address(a[i][j])=BaseAddress+(i×NumCols+j)×SizeOfType`

## 2D Arrays and Pointers

A 2D array in C is viewed as an **array of 1D arrays**, each row is a 1D array, and the array name is a pointer to the first row.

### Row as a Pointer

For `int A[m][n]`:​

- `A[0]` → address of **row 0** (pointer to first element of row 0)
- `A[1]` → address of **row 1**
- `A[2]` → address of **row 2**

So to access element `A[0][2]`:

```c
A[0][2] == *(A[0] + 2)
```

**General Rule:**

```c
A[i][j] == *(A[i] + j)
```

## Full Pointer Arithmetic Breakdown

For a Multi-dimensional array, pointer arithmetic extends like this:​

|        Expression        |                   Meaning                   |
| :----------------------: | :-----------------------------------------: |
|         `a + i`          |       Address of the `i`-th 2D array        |
|        `*(a + i)`        | Pointer to first element of `i`-th 2D array |
|      `*(a + i) + j`      |  Address of `j`-th row in `i`-th 2D array   |
|    `*(*(a + i) + j)`     |    Pointer to first element of that row     |
|  `*(*(a + i) + j) + k`   |    Address of `k`-th element in that row    |
| `*(*(*(a + i) + j) + k)` |          **Value** at `a[i][j][k]`          |

**Flattening a 2D Array with a Pointer**
Since C stores 2D arrays as a **single contiguous block**, you can pass a pointer to the first element and navigate the entire array using: `Address(A[i][j])=BaseAddress+(i×n+j)×SizeOfType`

```c
#include <stdio.h>

void print(int *arr, int m, int n) {
    for (int i = 0; i < m; i++) {
        for (int j = 0; j < n; j++) {
            printf("%d ", *(arr + i * n + j));  // flatten: row*cols + col
        }
        printf("\n");
    }
}

int main() {
    int arr[][3] = {{1, 2, 3},
                    {4, 5, 6},
                    {7, 8, 9}};

    print(&arr[0][0], 3, 3);   // pass address of first element
    return 0;
}
```

**Output:**

```shell
1 2 3
4 5 6
7 8 9
```

## Passing Multidimensional Arrays to Functions

When passing a 2D array to a function, the **first dimension can be left blank**, the compiler takes care of the rows. However, **all other dimensions must be explicitly defined** so the function knows the memory layout.​

### Why Must Other Dimensions Be Specified?

Remember, a 2D array is just a **contiguous block of memory**. The function needs to know the number of columns to correctly calculate the address of each element using:​

`Address(A[i][j])=BaseAddress+(i×NumCols+j)×SizeOfType`

Without knowing `NumCols`, the function **cannot navigate the array correctly**.

### Function Signature Options

All three are valid, choose what keeps your code clearest:​

```c
int example(int a[][5], int rows);      // preferred — first dim blank
int example(int a[3][5], int rows);     // compiler ignores the 3 — helps programmer
int example(int *arr, int m, int n);    // flattened pointer method
```

### Method 1 - Standard 2D Array Parameter

```c
#include <stdio.h>

void print2D(int a[][3], int rows) {
    for (int i = 0; i < rows; i++) {
        for (int j = 0; j < 3; j++) {
            printf("%d ", a[i][j]);
        }
        printf("\n");
    }
}

int main() {
    int arr[3][3] = {{1, 2, 3},
                     {4, 5, 6},
                     {7, 8, 9}};

    print2D(arr, 3);
    return 0;
}
```

**Output:**

```shell
1 2 3
4 5 6
7 8 9
```

### Method 2 - Flattened Pointer (More Efficient)

```c
#include <stdio.h>

void print(int *arr, int m, int n) {
    for (int i = 0; i < m; i++) {
        for (int j = 0; j < n; j++) {
            printf("%d ", *(arr + i * n + j));
        }
        printf("\n");
    }
}

int main() {
    int arr[3][3] = {{1, 2, 3},
                     {4, 5, 6},
                     {7, 8, 9}};

    print(&arr[0][0], 3, 3);    // pass address of first element
    return 0;
}
```

**Output:**

```shell
1 2 3
4 5 6
7 8 9
```

## Memory Layout of C Programs

When a C program runs, the OS divides its memory into **five distinct segments**, each serving a specific purpose.​

### The Five Segments

```text
High Address  ┌───────────────────┐
              │       Stack       │ ← grows downward
              ├───────────────────┤
              │         ↓         │
              │                   │
              │         ↑         │
              ├───────────────────┤
              │       Heap        │ ← grows upward
              ├───────────────────┤
              │    BSS Segment    │ ← uninitialized globals/statics
              ├───────────────────┤
              │   Data Segment    │ ← initialized globals/statics
              ├───────────────────┤
Low Address   │   Text Segment    │ ← compiled program code
              └───────────────────┘
```

### 1. Text Segment (Code Segment)

- Contains the **compiled executable instructions** of the program​
- **Read-only** — this is why string literals cannot be modified!
- Shared between processes running the same program

```c
int main() {
    printf("Hello\n");   // this instruction lives in the Text Segment
}
```

### 2. Data Segment (Initialized)

- Stores **global and static variables** that are **explicitly initialized** by the programmer​

```c
int x = 10;              // initialized global — lives in Data Segment
static int count = 5;    // initialized static — lives in Data Segment
```

## 3. BSS Segment (Uninitialized)

- Stores **global and static variables** that are **uninitialized or initialized to zero**​
- The **kernel automatically sets** all BSS variables to `0` before the program starts
- BSS stands for **"Block Started by Symbol"** — an ancient assembler term​

```c
int y;                   // uninitialized global → BSS, set to 0
static int arr[100];     // uninitialized static array → BSS, all zeros
```

## 4. Stack

- Stores **automatic (local) variables** and function call information​
- Follows **LIFO** (Last In, First Out) structure
- Each function call creates a **stack frame** containing:
  - Local variables
  - Return address
  - Saved register values
- Grows **downward** toward lower addresses on x86 architecture​
- Automatically managed — variables are created on entry and destroyed on exit

```c
void foo() {
    int a = 5;       // local variable — lives on the Stack
    float b = 3.14;  // destroyed when foo() returns
}
```

## 5. Heap

- Used for **dynamic memory allocation** at runtime​
- Grows **upward** toward higher addresses
- Managed manually using library functions:

```c
#include <stdlib.h>

int *p = malloc(100 * sizeof(int));   // allocate on Heap
free(p);                               // must free manually — or memory leak!
```

## Where Does Each Variable Live?

```c
int globalVar = 10;          // Data Segment  — initialized global
int uninitGlobal;            // BSS Segment   — uninitialized global

int main() {
    int localVar = 5;        // Stack         — local/automatic variable
    static int s = 3;        // Data Segment  — initialized static
    int *p = malloc(4);      // Heap          — dynamically allocated
    free(p);
}
```
