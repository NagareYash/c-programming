# Advanced Pointers, Function Pointers, and Input Handling

## Pointer Arithmetic

We know that a pointer stores a memory address. Since addresses are numeric values, C lets you perform arithmetic on pointers using `++`, `--`, `+`, and `-`. However, pointer arithmetic is **not** the same as regular integer arithmetic — it takes the **size of the data type** into account.
### How It Works

When you increment a pointer, it doesn't just add `1` to the address — it moves forward by `sizeof(type)` bytes:
```c
pointer + n  →  address + (n × sizeof(type))
```

**Example — `int` pointer (4 bytes each):**
```c
int arr[5] = {10, 20, 30, 40, 50};
int *p = arr;    // p points to arr[0] at address 1000

// p + 0 → 1000 (arr[0])
// p + 1 → 1004 (arr[1])
// p + 2 → 1008 (arr[2])
```

**Example — `char` pointer (1 byte each):**
```c
char str[] = "Hello";
char *cp = str;   // cp points to str[0] at address 2000

// cp + 0 → 2000 ('H')
// cp + 1 → 2001 ('e')
// cp + 2 → 2002 ('l')
```
### Increment and Decrement
```c
#include <stdio.h>

int main() {
    int arr[5] = {10, 20, 30, 40, 50};
    int *p = arr;

    printf("*p = %d\n", *p);       // 10
    p++;                             // move to next int (4 bytes forward)
    printf("*p = %d\n", *p);       // 20
    p++;
    printf("*p = %d\n", *p);       // 30
    p--;                             // move back one int (4 bytes backward)
    printf("*p = %d\n", *p);       // 20

    return 0;
}
```

**Output:**
```shell
*p = 10
*p = 20
*p = 30
*p = 20
```
### Visualizing Pointer Movement
```text
Address:   1000    1004    1008    1012    1016
         ┌───────┬───────┬───────┬───────┬───────┐
         │  10   │  20   │  30   │  40   │  50   │
         └───────┴───────┴───────┴───────┴───────┘
              ↑
              p (start)

After p++:   was     now
              ↓       ↓
         │  10   │  20   │  30   │  40   │  50   │
                     ↑
                     p (now at 1004)

After p--:
                     ↑
                     p (back to 1004)
```
### Step Size Depends on Data Type

| Data Type | `sizeof` | `p++` Moves By |
| --------- | -------- | --------------- |
| `char`    | 1 byte   | +1 byte         |
| `int`     | 4 bytes  | +4 bytes        |
| `float`   | 4 bytes  | +4 bytes        |
| `double`  | 8 bytes  | +8 bytes        |
### Adding/Subtracting an Integer

You can jump forward or backward by any number of elements:
```c
int arr[5] = {10, 20, 30, 40, 50};
int *p = arr;

printf("%d\n", *(p + 3));   // 40 — jumps 3 ints (12 bytes) forward
printf("%d\n", *(p + 4));   // 50 — jumps 4 ints (16 bytes) forward
```
### Pointer Subtraction

Subtracting two pointers of the **same type** gives the **number of elements** between them, not the number of bytes:
```c
int arr[5] = {10, 20, 30, 40, 50};
int *start = &arr[0];
int *end   = &arr[4];

printf("Elements apart: %ld\n", end - start);   // 4 (not 16!)
```

## Arrays of Pointers

In C, you can create an array where each element is a **pointer**. This is especially useful for managing a **list of strings**, since each pointer can point to a string of a different length.
### Why Arrays of Pointers?

Consider storing a list of names. With a 2D char array, every row must be the same size — wasting memory on shorter strings:
```c
// 2D array — every row is 20 chars, even short names waste space
char names[4][20] = {"John Doe", "Jane Smith", "Steve Smith", "Joseph Todd"};
```

With an **array of pointers**, each pointer points to only as much memory as that string needs:
```c
char str1[] = "John Doe";
char str2[] = "Jane Smith";
char str3[] = "Steve Smith";
char str4[] = "Joseph Todd";

char *ptr[4];
ptr[0] = str1;
ptr[1] = str2;
ptr[2] = str3;
ptr[3] = str4;

printf("%s\n", ptr[2]);   // Steve Smith
```
### Visualizing in Memory
```text
ptr[0] ──→ "John Doe\0"       (9 bytes)
ptr[1] ──→ "Jane Smith\0"     (11 bytes)
ptr[2] ──→ "Steve Smith\0"    (12 bytes)
ptr[3] ──→ "Joseph Todd\0"    (12 bytes)
```

Each pointer stores the **address** of the first character of its string — no wasted space.
### Swapping Pointers vs Swapping Data

Say you want to swap the first two names. You have **two choices**:

**Bad way — swap the actual string data:**

- Copy `str1` to a temp buffer character by character
- Copy `str2` into `str1`'s location
- Copy temp into `str2`'s location
- **Problem:** If `str2` is longer than `str1`, copying it over will write past `str1`'s memory — **crash!**

**Good way — swap the pointers:**

Just swap where `ptr[0]` and `ptr[1]` point to. The actual strings never move.
```c
#include <stdio.h>

int main() {
    char str1[] = "John Doe";
    char str2[] = "Jane Smith";
    char str3[] = "Steve Smith";
    char str4[] = "Joseph Todd";

    char *ptr[4], *swap;
    ptr[0] = str1;
    ptr[1] = str2;
    ptr[2] = str3;
    ptr[3] = str4;

    printf("Before: %s  %s\n", ptr[0], ptr[1]);

    // Swap pointers — not the data!
    swap   = ptr[0];
    ptr[0] = ptr[1];
    ptr[1] = swap;

    printf("After:  %s  %s\n", ptr[0], ptr[1]);

    return 0;
}
```

**Output:**
```shell
Before: John Doe  Jane Smith
After:  Jane Smith  John Doe
```

### What Actually Happened
```text
Before swap:                        After swap:
ptr[0] ──→ "John Doe"              ptr[0] ──→ "Jane Smith"
ptr[1] ──→ "Jane Smith"            ptr[1] ──→ "John Doe"
```

The strings stayed exactly where they were in memory — only the pointers changed. This is **faster** (swapping two addresses instead of copying entire strings) and **safer** (no risk of buffer overflow).
### Why This Matters

When sorting a list of strings, you **never** move the actual string data — you only rearrange the pointers:
1. **Fast** — swapping two 8-byte pointers vs copying potentially hundreds of characters
2. **Safe** — no risk of writing past array boundaries
3. **How real sorting works** — `qsort` on strings does exactly this (we'll see this later)

## Function Pointers

We know pointers can point to variables. In C, pointers can also point to **functions**. A function pointer stores the **address of a function's executable code** in memory, allowing you to call that function indirectly.
### Declaration

**Syntax:**
```c
return_type (*pointer_name)(parameter_types);
```

The parentheses around `*pointer_name` are **critical** — without them, the meaning changes entirely:
```c
void (*fp)(int);    // fp is a POINTER to a function that takes int, returns void
void *fp(int);      // fp is a FUNCTION that takes int, returns pointer to void — NOT what we want!
```
### Assigning and Calling

The **name of a function** is a pointer to its code — just like the name of an array is a pointer to its first element.
```c
#include <stdio.h>

void Hi_function(int times);

int main() {
    void (*function_ptr)(int);       // declare function pointer
    function_ptr = Hi_function;      // assign — no & needed!
    function_ptr(3);                 // call through pointer

    return 0;
}

void Hi_function(int times) {
    for (int k = 0; k < times; k++) {
        printf("Hi\n");
    }
}
```

**Output:**
```shell
Hi
Hi
Hi
```

**Note:** Since a function name is already an address, writing `function_ptr = Hi_function` and `function_ptr = &Hi_function` are both valid and equivalent.
### Passing Function Pointers as Arguments (Callbacks)

A **callback** is when you pass a function pointer to another function, letting the called function decide **which behavior to execute**. This makes your code more flexible — you can change behavior without modifying the function itself.
```c
#include <stdio.h>

void print();
void execute(void (*fun)());

int main() {
    execute(&print);     // pass address of print
    return 0;
}

void print() {
    printf("Hello!\n");
}

void execute(void (*fun)()) {
    fun();               // call whatever function was passed in
}
```

**Output:**
```shell
Hello!
```
### A More Practical Callback Example

Here, the same `add` function can output its result in **different ways** depending on which callback you pass:
```c
#include <stdio.h>
#include <stdlib.h>

int add(int a, int b, void (*print_callback)(int));
void console_print(int value);
void loud_print(int value);

int main() {
    add(10, 20, console_print);    // prints normally
    add(10, 20, loud_print);       // prints with emphasis

    return 0;
}

int add(int a, int b, void (*print_callback)(int)) {
    int sum = a + b;
    print_callback(sum);           // call whichever print function was passed
    return sum;
}

void console_print(int value) {
    printf("Result: %d\n", value);
}

void loud_print(int value) {
    printf("*** RESULT: %d ***\n", value);
}
```

**Output:**
```shell
Result: 30
*** RESULT: 30 ***
```

The power here is that `add` doesn't need to know **how** the result is printed — it just calls whatever function you give it. You can add new print functions without changing a single line of `add`.
## Array of Function Pointers

Just like arrays of `int` or `char *`, you can create an **array of function pointers**. This acts as a **dispatch table** — replacing long `if-else` or `switch` chains with a simple array lookup.
### Declaration
```c
return_type (*array_name[size])(parameter_types);
```
### Example — Calculator Without switch
```c
#include <stdio.h>

int sum(int x, int y)  { return x + y; }
int sub(int x, int y)  { return x - y; }
int mult(int x, int y) { return x * y; }
int divide(int x, int y) {
    if (y != 0) return x / y;
    else return 0;
}

int main() {
    int x, y, choice, result;

    // Array of 4 function pointers
    int (*ope[4])(int, int);
    ope[0] = sum;
    ope[1] = sub;
    ope[2] = mult;
    ope[3] = divide;

    printf("Enter two integer numbers: ");
    scanf("%d %d", &x, &y);

    printf("Enter 0 to sum, 1 to subtract, 2 to multiply, or 3 to divide: ");
    scanf("%d", &choice);

    result = ope[choice](x, y);   // call the function at index 'choice'
    printf("Result: %d\n", result);

    return 0;
}
```

**Output:**
```shell
Enter two integer numbers: 10 5
Enter 0 to sum, 1 to subtract, 2 to multiply, or 3 to divide: 2
Result: 50
```
### How It Works
```text
ope[0] ──→ sum()       "Enter 0 to sum"
ope[1] ──→ sub()       "Enter 1 to subtract"
ope[2] ──→ mult()      "Enter 2 to multiply"
ope[3] ──→ divide()    "Enter 3 to divide"
```

Instead of a `switch` with 4 cases, `ope[choice](x, y)` directly calls the right function. This scales much better — adding a new operation means just writing a new function and adding it to the array.
### Comparison — switch vs Dispatch Table

|                      | `switch` Statement     | Array of Function Pointers         |
| -------------------- | ---------------------- | ---------------------------------- |
| Adding new operation | Add a new `case` block | Add one function + one array entry |
| Code length          | Grows with each case   | Stays compact                      |
| Readability          | Good for small menus   | Cleaner for many options           |
| Flexibility          | Fixed at compile time  | Can be changed at runtime          |
## `qsort` — Sorting with the Standard Library

C provides a built-in sorting function called `qsort()` in `<stdlib.h>`. Instead of writing your own sorting algorithm, you can use `qsort` to sort **any type of array** — `int`, `float`, strings, or even custom data types. The key is that **you** provide a **comparator function** that tells `qsort` how to compare two elements.

### Function Prototype
```c
void qsort(void *base, size_t nitems, size_t size, int (*compar)(const void *, const void *));
```

| Parameter | Meaning |
| --------- | ------- |
| `base`    | Pointer to the first element of the array |
| `nitems`  | Number of elements in the array |
| `size`    | Size of each element in bytes — use `sizeof(arr[0])` |
| `compar`  | Pointer to a comparator function |
### The Comparator Function

The comparator receives two `void *` pointers (generic pointers that can point to any type). You must **cast** them to the correct type and compare:
```c
int compare(const void *a, const void *b) {
    int x = *(const int *)a;    // cast void* to int*, then dereference
    int y = *(const int *)b;

    if (x < y) return -1;       // a comes before b
    if (x > y) return  1;       // a comes after b
    return 0;                    // a and b are equal
}
```
### Why `void *`?

`qsort` is designed to sort **any data type**. Since it doesn't know in advance whether you're sorting `int`, `float`, or strings, it uses `void *` — a generic pointer. The comparator function is where **you** tell it the actual type by casting.

### Required Return Values

| Return Value | Meaning |
| ------------ | ------- |
| Negative (e.g. `-1`) | First element goes **before** second |
| `0` | Elements are **equal** |
| Positive (e.g. `1`) | First element goes **after** second |
### Example — Sorting Integers in Ascending Order
```c
#include <stdio.h>
#include <stdlib.h>

int compare_asc(const void *a, const void *b) {
    int x = *(const int *)a;
    int y = *(const int *)b;

    if (x < y) return -1;
    if (x > y) return  1;
    return 0;
}

int main() {
    int arr[] = {42, 7, 99, 3, 15};
    int n = sizeof(arr) / sizeof(arr[0]);

    printf("Before: ");
    for (int i = 0; i < n; i++) printf("%d ", arr[i]);
    printf("\n");

    qsort(arr, n, sizeof(arr[0]), compare_asc);

    printf("After:  ");
    for (int i = 0; i < n; i++) printf("%d ", arr[i]);
    printf("\n");

    return 0;
}
```

**Output:**
```shell
Before: 42 7 99 3 15
After:  3 7 15 42 99
```
### Changing Sort Order

The beauty of `qsort` is that **changing the comparator changes the behavior** — the rest of the code stays the same.

**Ascending:** return `-1` when `a < b`
```c
if (x < y) return -1;
if (x > y) return  1;
```

**Descending:** flip the logic — return `-1` when `a > b`
```c
if (x > y) return -1;
if (x < y) return  1;
```
### Passing the Comparator

Since the function name **is** a pointer to the function, you don't need `&`:
```c
qsort(arr, n, sizeof(arr[0]), compare_asc);    // both are valid
qsort(arr, n, sizeof(arr[0]), &compare_asc);   // same thing
```
### ⚠️ Common Mistake — Using `return a - b`

You might see comparators written as:
```c
return *(int *)a - *(int *)b;   // shortcut — but DANGEROUS!
```

This works most of the time, but can cause **undefined behavior** with large numbers due to **signed integer overflow**. For example, if `a = 2000000000` and `b = -2000000000`, the subtraction overflows. Always use explicit `if` comparisons — it's safer.

## Advanced `scanf` for Strings

We know that `scanf("%s", str)` reads a string — but it **stops at the first whitespace**. This means you can't read a full sentence like `"Hello World"` with `%s`. C provides special format specifiers inside `scanf` to control **exactly which characters** are accepted.

### `%[...]` — Include Set

The `%[...]` specifier tells `scanf` to **only accept** characters listed inside the brackets. Input stops as soon as a character **not in the set** is encountered.
```c
char str[100];
scanf("%[aeiou]", str);    // only accepts vowels — stops at first non-vowel
```

**Input:** `aeiouXYZ`
**Result:** `str = "aeiou"` — stopped at `X`

### `%[^...]` — Exclude Set

The `%[^...]` specifier accepts **everything except** the characters listed after `^`. This is far more useful in practice.
```c
char str[100];
scanf("%[^\n]", str);      // accepts everything EXCEPT newline
```
This effectively reads an **entire line** including spaces — something `%s` cannot do.

**Example:**
```c
#include <stdio.h>

int main() {
    char str[100];

    printf("Enter a sentence: ");
    scanf("%[^\n]", str);

    printf("You entered: %s\n", str);

    return 0;
}
```

**Output:**
```shell
Enter a sentence: Hello World 123
You entered: Hello World 123
```

### The Buffer Problem

When you use `scanf` multiple times, a leftover `\n` (from pressing Enter) can remain in the input buffer. This causes the next `scanf` to read that `\n` immediately and terminate.

**Solution:** Flush the input buffer between reads:
```c
int i;
char str[100];

printf("Enter an integer: ");
scanf("%d", &i);

fflush(stdin);              // clear the leftover \n from buffer

printf("Enter a string: ");
scanf("%[^\n]", str);       // now this works correctly
```

**Without `fflush(stdin)`**, the second `scanf` would see the leftover `\n` and store an empty string.

### `%[^...]` with Other Terminators

You can exclude **any characters**, not just newline:
```c
char str[100];
scanf("%[^ABC]", str);     // stops at A, B, or C — everything else is accepted
```

**Input:** `hello world A test`
**Result:** `str = "hello world "` — stopped at `A`

## `fgets`, `puts`, and `fputs`

### `fgets` — Safer String Input

`fgets()` is the **safest way** to read a string in C. Unlike `scanf`, it lets you specify a **maximum length**, preventing buffer overflow.

**Syntax:**
```c
char *fgets(char *str, int n, FILE *stream);
```

| Parameter | Meaning                                     |
| --------- | ------------------------------------------- |
| `str`     | Character array to store input              |
| `n`       | Maximum characters to read (including `\0`) |
| `stream`  | Input source — use `stdin` for keyboard     |
`fgets` stops when it reads `n-1` characters, hits a newline `\n`, or reaches end-of-file (EOF) — whichever comes first.

**Example:**
```c
#include <stdio.h>

int main() {
    char str[11];

    printf("Enter a string: ");
    fgets(str, 10, stdin);

    printf("You entered: %s\n", str);

    return 0;
}
```

**Note:** `fgets` **includes** the newline `\n` in the string if there's room. This is different from `scanf` which discards it.
### `puts` — Simple String Output

`puts()` writes a string to the screen and **automatically adds a newline** at the end.
```c
#include <stdio.h>

int main() {
    char str[] = "Hello, World!";
    puts(str);      // prints: Hello, World!\n — newline added automatically

    return 0;
}
```

**Output:**
```shell
Hello, World!
```
### `fputs` — Output Without Automatic Newline

`fputs()` writes a string to a specified stream (screen, file, etc.) **without** adding a newline.

**Syntax:**
```c
int fputs(const char *str, FILE *stream);
```
```c
#include <stdio.h>

int main() {
    fputs("Hello ", stdout);     // no newline added
    fputs("World!\n", stdout);   // manually add \n

    return 0;
}
```

**Output:**
```shell
Hello World!
```
### Comparison — String I/O Functions

| Function | Reads/Writes | Adds `\n`? | Buffer Safe? | Best For |
| -------- | ------------ | ---------- | ------------ | -------- |
| `scanf("%s")` | Reads | No | No | Single words |
| `scanf("%[^\n]")` | Reads | No | No | Full lines (with caution) |
| `fgets()` | Reads | Keeps existing `\n` | **Yes** | Safest string input |
| `printf()` | Writes | Manual | N/A | Formatted output |
| `puts()` | Writes | **Yes** (automatic) | N/A | Simple string output |
| `fputs()` | Writes | No | N/A | Output to any stream |
### Best Practice

The safest combination for string I/O is `fgets` for input and `printf` for output:
```c
char name[50];

printf("Enter your name: ");
fgets(name, 50, stdin);        // safe — won't overflow

printf("Hello, %s", name);    // fgets keeps the \n, so no need to add one
```
