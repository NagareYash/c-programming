## Pointers — Declaration and Basics
All variables in a program are stored at a particular **memory location**. A **pointer** is a special variable that stores the **memory address** of another variable — it's an indirect way of accessing data.
### Declaring a Pointer

**Syntax:**
```c
type *pointer_name;
```

**Examples:**
```c
int *p;       // pointer to an integer
float *fp;    // pointer to a float
char *cp;     // pointer to a char
```

- The `*` in the declaration means "this is a pointer"
- A pointer **must be associated** with a variable before use — an uninitialized pointer is dangerous
### Key Operators

| Operator | Name        | Meaning                                      |
| -------- | ----------- | -------------------------------------------- |
| `&`      | Address-of  | Returns the **memory address** of a variable |
| `*`      | Dereference | Returns the **value at** a memory address    |

**Example:**
```c
#include <stdio.h>

int main() {
    int i, j;
    int *p;         // declare pointer to int

    i = 10;
    p = &j;         // p points to address of j
    *p = i;         // put value of i into address pointed to by p (j)

    printf("i = %d\n", i);       // 10
    printf("*p = %d\n", *p);     // 10
    printf("j = %d\n", j);       // 10 — same as *p

    return 0;
}
```
### Visualizing Memory
```shell
Variable    Address      Value
--------    -------      -----
   i        0x1000        10
   j        0x1004        10   ← *p writes here
   p        0x1008      0x1004  ← stores address of j
```

- `p` holds the **address** `0x1004`
- `*p` gives the **value at** `0x1004` which is `10`
- `&j` gives the **address** of `j` which is `0x1004`
### Using Pointer with `scanf`
```c
#include <stdio.h>

int main() {
    int i;
    int *p;

    p = &i;                  // p points to i

    printf("Enter a number: ");
    scanf("%d", p);          // no & needed — p is already an address!

    printf("%d\n", *p);      // dereference p to get value of i

    return 0;
}
```

**Note:** Since `p` is already an address, we don't write `&p` in `scanf` — that would be the "address of the address"!
## Dereferencing Pointers (* and &)
**Dereferencing** means accessing the **value stored at** the memory address a pointer is pointing to. This is done using the `*` operator.
### The Two Operators
**`&` — Address-of Operator:**
- Used to **get the address** of a variable
- Turns a variable into a pointer

```c
int x = 10;
printf("%p\n", &x);   // prints memory address of x e.g. 0x1000
```

**`*` — Dereference Operator:**
- Used to **get the value at** an address
- Turns a pointer back into a value

```c
int x = 10;
int *p = &x;
printf("%d\n", *p);   // prints 10 — value at address p
```

**Example - Alter and Transfer Variables:**
```c
#include <stdio.h>

int main() {
    float x, y;
    float *fp, *fp2;

    x = 6.2;

    // print value and address of x
    printf("Value of x: %f, Address of x: %ld\n", x, (long)&x);

    fp = &x;           // fp points to x

    // print value at fp (which is x)
    printf("Value at fp: %f\n", *fp);

    // change value at fp — this changes x!
    *fp = 9.2;
    printf("New value of x: %f\n", x);       // x is now 9.2

    // transfer values using pointers
    y   = *fp;         // y gets value at fp (9.2)
    fp2 = fp;          // fp2 now points to same address as fp

    printf("y = %f, *fp2 = %f\n", y, *fp2);  // both 9.2

    return 0;
}
```

**Output:**
```shell
Value of x: 6.200000, Address of x: 4201234
Value at fp: 6.200000
New value of x: 9.200000
y = 9.200000, *fp2 = 9.200000
```
### Visualizing What Happens
```text
Step 1: fp = &x
        fp ──────→ [ 6.2 ]  (address of x)
Step 2: *fp = 9.2
        fp ──────→ [ 9.2 ]  (x is now changed!)
Step 3: y = *fp
        y  = 9.2   (copy of value)
Step 4: fp2 = fp
        fp2 ─────→ [ 9.2 ]  (fp2 points to same address as fp)
```
### Common Mistakes

|Mistake|Problem|
|---|---|
|`int *p;` then use `*p` without assigning|Uninitialized pointer — **dangerous!**|
|`scanf("%d", &p)` when p is already a pointer|`&p` = address of the pointer itself — **wrong!**|
|`p = x` instead of `p = &x`|Assigning value instead of address — **wrong!**|
## Pointers in Functions (Pass by Reference)
We already know that **pass by value** gives a function a copy — changes don't affect the original. By passing a **pointer** (memory address) instead, the function can directly modify the original variable.
### The Problem — Pass by Value
```c
void BadSwap(int a, int b) {
    int tmp;
    tmp = a;
    a = b;
    b = tmp;    // only swaps local copies!
}

int main() {
    int a = 1, b = 2;
    BadSwap(a, b);
    printf("A: %d, B: %d\n", a, b);  // still 1, 2!
}
```
### Returning Multiple Values Using Pointers
A function can only `return` **one value**. Pointers let you effectively return **multiple values** at once by modifying variables directly.
```c
#include <stdio.h>

// Function Prototypes
int fun1(int *d, int *e);

int main() {
    int a, b, c;

    printf("Enter 2 numbers: ");
    scanf("%d %d", &a, &b);

    c = fun1(&a, &b);           // pass addresses of a and b
    printf("Result: %d\n", c);

    return 0;
}

int fun1(int *d, int *e) {
    int f;
    f = *d * (*e);              // dereference to multiply
    return f;
}
```

**Output:**
```shell
Enter 2 numbers: 5 4
Result: 20
```
## Rules to Remember
- Use `&variable` when **calling** a function to pass address
- Use `*pointer` inside the function to **access or modify** the value
- Use `*pointer` in the **parameter list** to declare it receives an address
- Never pass `&p` if `p` is already a pointer — that's the address of the pointer itself

## Arrays — Declaration and Initialization
Programs often need to store many values of the **same type**. Instead of declaring separate variables, arrays let you store a **fixed-size collection** under one name.
### Why Arrays?
```c
// Without arrays — messy!
int grade0, grade1, grade2, grade3;   // imagine 40 students...

// With arrays — clean!
int grade[40];                         // stores 40 integers
```
### Declaration
**Syntax:**
```c
type array_name[size];
```

**Examples:**
```c
int test[40];       // array of 40 integers
float scores[10];   // array of 10 floats
char name[20];      // array of 20 characters
```

- Array index always starts at **0**
- An array of size `n` has valid indices `0` to `n-1`
- `int grade[40]` gives you `grade[0]` to `grade[39]`
### Initialization
**Initialize each element separately:**
```c
float A[5] = {0.0, 1.0, 2.0, 3.0, 4.0};
int   B[3] = {1, 2, 3};
```

**Initialize to zero:**
```c
int a[100] = {0};       // all elements set to 0
```

**Let compiler count the size:**
```c
float var[] = {12.7, 18.9, 20.3, 45.88, 11.19};
// compiler automatically sets size to 5
```

**Using `#define` for size:**
```c
#define MAX 40
int a[MAX];             // cleaner, easier to change later
```
### ⚠️ Uninitialized Arrays
If you declare an array without initializing it, elements contain **random garbage values**:
```c
int w[20];              // elements have random values — dangerous!
```

Always initialize before use:
```c
int w[20] = {0};        // safe — all zeros
```
## Storage Keywords
**`auto` (Automatic) — Default:**
- Local variables are automatic by default
- Created when function is entered, destroyed when function exits

```c
void demo() {
    int x = 10;    // auto by default — destroyed when demo() exits
}
```

**`static`:**
- Variable **persists in memory** until the program ends
- Keeps its value between function calls
- Acts like a global but scoped to its local function

```c
void counter() {
    static int count = 0;   // retains value between calls!
    count++;
    printf("Count: %d\n", count);
}

int main() {
    counter();   // Count: 1
    counter();   // Count: 2
    counter();   // Count: 3
}
```

**Static Array:**
```c
static int a[100];   // entire array initialized to 0 automatically
```
## Accessing Array Elements (Subscripting)
To access an individual element of an array, use the **index** inside square brackets`[]`.
### Syntax
```c
array_name[index]
```

**Example:**
```c
int A[5] = {10, 20, 30, 40, 50};

printf("%d\n", A[0]);   // 10 — first element
printf("%d\n", A[2]);   // 30 — third element
printf("%d\n", A[4]);   // 50 — last element
```

**Memory Representation:**
```text
Index →    [0]   [1]   [2]   [3]   [4]
           ┌─────┬─────┬─────┬─────┬─────┐
A[]  →     │ 10  │ 20  │ 30  │ 40  │ 50  │
           └─────┴─────┴─────┴─────┴─────┘
```
### Reading and Writing Elements
```c
int val[20];

val[12] = 70;                    // write — store 70 at index 12
int temp = val[12];              // read  — copy value at index 12

scanf("%d", &val[12]);           // read from user — note the &
printf("%d\n", val[12]);         // print element at index 12
```
### Looping Through an Array
```c
#define MAX 5
int grades[MAX] = {85, 90, 78, 92, 88};
int sum = 0;
float avg;

for (int i = 0; i < MAX; i++) {
    sum += grades[i];
    printf("Grade[%d] = %d\n", i, grades[i]);
}

avg = (float)sum / MAX;
printf("Average: %.2f\n", avg);
```

**Output:**
```shell
Grade[0] = 85
Grade[1] = 90
Grade[2] = 78
Grade[3] = 92
Grade[4] = 88
Average: 86.60
```
## ⚠️ Common Mistakes

|Mistake|Problem|
|---|---|
|`A[5]` on a size-5 array|**Out of bounds** — valid indices are 0 to 4|
|`A[-1]`|**Invalid index** — no negative indices|
|Forgetting `&` in `scanf`|`scanf("%d", val[i])` → **wrong**, use `&val[i]`|
### Array Name as Address
The **name of an array** is actually a pointer to its **first element (index 0)**:
```c
int A[5] = {10, 20, 30, 40, 50};

printf("%p\n", A);      // address of A[0]
printf("%p\n", &A[0]);  // same address!
```

This becomes important in the **Arrays and Pointers** section later.
## Arrays as Function Arguments
When you pass an array to a function, C does **not** copy the entire array. Instead, it passes the **address of the first element** — meaning the function works directly on the original array.
### Key Rule
```c
void myFunc(int arr[], int size);   // arr is a pointer to the first element
```

- You **cannot** pass an array by value in C
- Changes made inside the function **affect the original array**
- You must **also pass the size** — the function has no way to know it otherwise

**Example - Print an Array:**
```c
#include <stdio.h>
#define MAX 5

void printArray(int arr[], int size) {
    for (int i = 0; i < size; i++) {
        printf("arr[%d] = %d\n", i, arr[i]);
    }
}

int main() {
    int nums[MAX] = {10, 20, 30, 40, 50};
    printArray(nums, MAX);   // pass array name (address) + size
    return 0;
}
```

**Output:**
```shell
arr[0] = 10
arr[1] = 20
arr[2] = 30
arr[3] = 40
arr[4] = 50
```
### Modifying an Array Inside a Function
Since the function receives the **actual address**, modifications persist after the function returns:
```c
#include <stdio.h>
#define MAX 5

void doubleAll(int arr[], int size) {
    for (int i = 0; i < size; i++) {
        arr[i] = arr[i] * 2;    // modifies original!
    }
}

int main() {
    int nums[MAX] = {1, 2, 3, 4, 5};
    
    doubleAll(nums, MAX);

    for (int i = 0; i < MAX; i++) {
        printf("%d ", nums[i]);  // 2 4 6 8 10
    }
    return 0;
}
```
### Equivalent Function Signatures
These three declarations are **identical** in C — all receive a pointer:
```c
void func(int arr[], int size);      // most readable — preferred
void func(int *arr, int size);       // explicit pointer notation
void func(int arr[MAX], int size);   // size is ignored by compiler
```

**Example - Sum and Average:**
```c
#include <stdio.h>
#define MAX 5

float calcAverage(int arr[], int size) {
    int sum = 0;
    for (int i = 0; i < size; i++) {
        sum += arr[i];
    }
    return (float)sum / size;
}

int main() {
    int grades[MAX] = {85, 90, 78, 92, 88};
    float avg = calcAverage(grades, MAX);
    printf("Average: %.2f\n", avg);   // Average: 86.60
    return 0;
}
```
### Pass by Value vs. Array Argument

| Behavior                 | Regular Variable | Array                    |
| ------------------------ | ---------------- | ------------------------ |
| What's passed            | Copy of value    | Address of first element |
| Changes affect original? | No               | Yes                      |
| Need to pass size?       | No               | Yes                      |
## Arrays and Pointers
An array name in C is actually a **pointer to its first element**. This means arrays and pointers are deeply intertwined, you can use them almost interchangeably.​

### Array Name = Pointer to First Element
```c
int A[5] = {10, 20, 30, 40, 50};

printf("%p\n", A);      // address of A[0]
printf("%p\n", &A[0]);  // same address!
```

- `A` is a pointer to `A[0]` — the **base address** of the array​
- You do **not** use `&` when assigning an array to a pointer (it's already an address)
### Creating a Pointer to an Array
```c
int A[100];
int *Apointer;

Apointer = A;       // point to start of array — NO & needed!
// equivalent to:
Apointer = &A[0];   // same thing
```

Now `Apointer` can be used to access any element of the array.​
### Subscript vs. Pointer Notation
These two expressions are **completely equivalent**:​
```c
A[i]    ==    *(A + i)
```

**Example - two ways to write the same thing:**
```c
int A[5] = {10, 20, 30, 40, 50};
int *p = A;

printf("%d\n", A[2]);     // 30 — subscript notation
printf("%d\n", *(p + 2)); // 30 — pointer notation
printf("%d\n", p[2]);     // 30 — pointers can be subscripted too!
```

To put value `7` into the 5th element:
```c
A[4] = 7;      // subscript way
*(A + 4) = 7;  // pointer way — identical result
```
### Traversing an Array with a Pointer
Using a pointer and incrementing through the array is **more computationally efficient** when processing elements sequentially:​
```c
#include <stdio.h>
#define MAX 5

int main() {
    double balance[MAX] = {1000.0, 2.0, 3.4, 17.0, 50.0};
    double *p = balance;    // p points to balance[0]

    printf("Using pointer:\n");
    for (int i = 0; i < MAX; i++) {
        printf("p[%d] = %.1f\n", i, *(p + i));
    }

    printf("Using array name directly:\n");
    for (int i = 0; i < MAX; i++) {
        printf("balance[%d] = %.1f\n", i, balance[i]);
    }
    return 0;
}
```
### Passing Array Elements as Pointers
A specific array element can be passed as a pointer to a function:​
```c
#include <stdio.h>

void swap(int *in, int *out) {
    int tmp = *in;
    *in = *out;
    *out = tmp;
}

int main() {
    int a[10];
    a[5] = 44;
    a[6] = 55;

    printf("%d %d\n", a[5], a[6]);   // 44 55
    swap(&a[5], &a[6]);               // pass elements as pointers
    printf("%d %d\n", a[5], a[6]);   // 55 44
    return 0;
}
```

**Example - Sine Array Using Pointer:**
```c
#include <stdio.h>
#include <math.h>

void sinearray(float *s) {
    float dr;
    float *arrayptr = s;          // pointer to start of array

    for (int j = 0; j < 180; j++) {
        dr = (float)j * 0.0174;   // convert degrees to radians
        *(arrayptr + j) = sin(dr);// store sine value via pointer
    }
}

int main() {
    float sarray[180];
    sinearray(sarray);

    for (int i = 0; i < 180; i++) {
        printf("%d: %.4f\n", i, sarray[i]);
    }
    return 0;
}
```
### Key Takeaways

| Expression | Meaning                         |
| ---------- | ------------------------------- |
| `A`        | Address of first element `A[0]` |
| `*A`       | Value at `A[0]`                 |
| `A + i`    | Address of `A[i]`               |
| `*(A + i)` | Value at `A[i]`                 |
| `&A[i]`    | Address of `A[i]`               |

**Why use pointer arithmetic?** When iterating **sequentially**, incrementing a pointer (`p++`) avoids the subscript index recalculation and can be faster in performance-critical code.

## String Literals vs Character Arrays
This is a critical distinction:​
```c
char *cards = "JQK";    // pointer to string LITERAL — READ ONLY
char cards[] = "JQK";   // char array — modifiable
```

- A **string literal** (`"JQK"`) is loaded into **read-only memory** by the OS
- Trying to modify it via a pointer causes a **crash or undefined behavior**
- Declaring it as a `char[]` **copies** the literal into stack memory which is safe to modify
